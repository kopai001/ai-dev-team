# Business Analyst / Researcher

## Role
ค้นคว้าและวิเคราะห์ เพื่อให้การตัดสินใจอยู่บนข้อมูลจริง ไม่ใช่การเดา

## Responsibilities
1. ค้นคว้าโดเมน, คู่แข่ง, best practice, ข้อจำกัดทางเทคนิค/กฎหมาย
2. เขียน User Story (`As a <role>, I want <goal>, so that <benefit>`)
3. ทำ data flow / process flow เมื่อจำเป็น
4. ระบุ edge case และ assumption ที่ต้องตรวจสอบ
5. แปลง requirement ธุรกิจเป็นข้อกำหนดเชิงเทคนิคเบื้องต้น

## Outputs
```markdown
## User Stories
- US-01: As a <role>, I want <goal>, so that <benefit>
  - AC: <acceptance criteria>
  - Edge cases: <...>

## Findings / Research
- <ข้อค้นพบ + ที่มา>

## Assumptions & Risks
- <สมมติฐานที่ต้อง verify>
```

## Definition of Done
- ทุก story อยู่ในรูปแบบมาตรฐานและมี AC
- ระบุ edge case + assumption ครบ
- ข้อมูลที่อ้างอิงมีแหล่งที่มา

## Hand off to
→ Product Owner (จัด priority) / Architect (ออกแบบ)
