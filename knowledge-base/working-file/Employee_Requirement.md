## 1. วัตถุประสงค์ของโมดูล

ระบบจัดการข้อมูลพนักงาน ถูกออกแบบมาเพื่อเป็น **ศูนย์กลางข้อมูล** ของพนักงานทั้งหมดในองค์กร โดยรวบรวมข้อมูลส่วนบุคคล โครงสร้างองค์กร ตำแหน่งงาน เอกสารสำคัญ ผู้ติดต่อฉุกเฉิน และข้อมูลสุขภาพ ไว้ในที่เดียว เพื่อใช้เป็นข้อมูลพื้นฐานเชื่อมโยงไปยังระบบงานอื่นๆ ต่อไป

### 1.1 เป้าหมายหลัก

1. เป็นข้อมูลพื้นฐานเชื่อมโยงไปยังระบบงานอื่นๆ (Time Tracking, Leave Management, Attendance, Payroll)
2. ลดความซ้ำซ้อนและความขัดแย้งของข้อมูลในองค์กร
3. เพิ่มความปลอดภัยและการควบคุมการเข้าถึงข้อมูล
4. รองรับมาตรฐานการคุ้มครองข้อมูลส่วนบุคคล (PDPA)

## 2. ขอบเขตของโมดูล

| :---: | :------------------- | :--------------------- |
|   1   | ข้อมูลพื้นฐานพนักงาน | Employee Profile       |
|   2   | ข้อมูลส่วนบุคคล      | Personal Information   |
|   3   | ผู้ติดต่อฉุกเฉิน     | Emergency Contacts     |
|   4   | ข้อมูลสุขภาพ         | Health Records         |
|   5   | เอกสารและใบรับรอง    | Document List          |
|   6   | โครงสร้างองค์กร      | Organization Structure |
|   7   | ระดับตำแหน่งงาน      | Position Level         |

## 3. รายละเอียดฟังก์ชันการทำงาน

### 3.1 โครงสร้างองค์กร (Organization Structure)

ระบบรองรับการตั้งค่าโครงสร้างองค์กรแบบ **3 ระดับ** ตามมาตรฐานการบริหารองค์กรขนาดใหญ่ โดยมีลำดับชั้นดังนี้

```
ระดับที่ 1: ฝ่าย (Division)
    └── ระดับที่ 2: แผนก (Department)
            └── ระดับที่ 3: หน่วย (Unit)
```

#### คุณสมบัติหลัก

> **ข้อสำคัญ:** พนักงานทุกคนต้องสังกัดถึงระดับหน่วย (Unit) เสมอ

- ผู้ดูแลระบบสามารถเพิ่ม แก้ไข หรือปิดการใช้งานฝ่าย แผนก และหน่วยได้
- รองรับชื่อทั้งภาษาไทยและภาษาอังกฤษ
- แสดงผลในรูปแบบ Tree View เห็นภาพรวมทั้งโครงสร้างองค์กร
- แสดงจำนวนพนักงานในแต่ละหน่วยงานแบบเรียลไทม์
- ระบบป้องกันการลบหน่วยงานที่ยังมีพนักงานสังกัดอยู่
- การยุบหน่วยงานต้องย้ายพนักงานไปหน่วยอื่นก่อนจึงจะสามารถปิดได้

---

### 3.2 ระดับตำแหน่งงาน (Position Level)

ระบบใช้แนวคิดตัวเลขลำดับชั้นเพื่อกำหนดอำนาจการบริหาร โดยมีหลักการคือ ตัวเลขน้อยหมายถึงอำนาจสูง

| ระดับ | ความหมาย     |
| :---: | :----------- |
|   0   | ผู้บริหาร    |
|  10   | กรรมการ      |
|  20   | หัวหน้าฝ่าย  |
|  30   | หัวหน้าแผนก  |
|  40   | หัวหน้าหน่วย |
|  99   | พนักงาน      |

#### คุณสมบัติหลัก

- ผู้ดูแลระบบสามารถเพิ่ม / แก้ไข ระดับตำแหน่งได้
- รองรับการสร้างตำแหน่งงานหลายตำแหน่งภายในระดับเดียวกัน
- พนักงานในระดับเดียวกัน สามารถสังกัดต่างฝ่าย แผนก หน่วย กันได้

---

### 3.3 ข้อมูลพื้นฐานพนักงาน (Employee Profile)

ข้อมูลพื้นฐานเป็นข้อมูลหลักที่ใช้อ้างอิงพนักงานในระบบทั้งหมด ประกอบด้วยข้อมูลหลักดังนี้

| ฟิลด์ข้อมูล    | รายละเอียด                           |
| :------------- | :----------------------------------- |
| รหัสพนักงาน    | สร้างอัตโนมัติตามรูปแบบที่ตั้งค่าไว้ |
| สถานะการทำงาน  | Active / Inactive / On Leave         |
| วันที่เริ่มงาน | วันที่เริ่มต้นทำงานในองค์กร          |
| วันที่ลาออก    | วันที่สิ้นสุดการทำงาน (ถ้ามี)        |
| สังกัด         | ฝ่าย → แผนก → หน่วย                  |
| ตำแหน่งงาน     | ตำแหน่งและระดับตำแหน่ง               |
| ประเภทพนักงาน  | Full-time / Part-time / Contract     |

#### คุณสมบัติหลัก

- ค้นหาพนักงานด้วยชื่อ รหัส หรือเลขประจำตัวประชาชน
- กรองข้อมูลตามฝ่าย แผนก หน่วย ตำแหน่ง และสถานะ
- ส่งออกข้อมูลพนักงานเป็นไฟล์ Excel
- นำเข้าข้อมูลพนักงานครั้งละหลายคน (Bulk Import)

---

### 3.4 ข้อมูลส่วนบุคคล (Personal Information)

ข้อมูลส่วนบุคคลแบ่งออกเป็น 3 กลุ่มข้อมูล เพื่อให้ง่ายต่อการจัดการและการเข้าถึง

#### กลุ่มที่ 1: ข้อมูลประจำตัว

- คำนำหน้าชื่อ ชื่อ-นามสกุล (ภาษาไทยและภาษาอังกฤษ)
- เลขบัตรประจำตัวประชาชน (จัดเก็บแบบเข้ารหัสตามมาตรฐาน PDPA)
- วันเดือนปีเกิด เพศ สัญชาติ ศาสนา
- สถานภาพสมรส
- รูปโปรไฟล์

#### กลุ่มที่ 2: ข้อมูลติดต่อ

- เบอร์โทรศัพท์มือถือ
- อีเมลส่วนตัว
- Facebook / Line
- ที่อยู่ปัจจุบัน

#### กลุ่มที่ 3: ข้อมูลธนาคาร (สำหรับ Payroll)

- ธนาคาร เลขที่บัญชี และชื่อบัญชี
- เลขประกันสังคม
- เลขประจำตัวผู้เสียภาษี

---

### 3.5 ผู้ติดต่อฉุกเฉิน (Emergency Contacts)

ระบบรองรับการบันทึกผู้ติดต่อฉุกเฉินสำหรับพนักงาน 1 คน ได้หลายราย เพื่อรองรับสถานการณ์ที่ไม่สามารถติดต่อบุคคลแรกได้

#### คุณสมบัติหลัก

- พนักงาน 1 คน สามารถเพิ่มผู้ติดต่อฉุกเฉินได้หลายคน
- กำหนดลำดับความสำคัญในการติดต่อ
- ระบุความสัมพันธ์ (พ่อ แม่ คู่สมรส พี่น้อง อื่นๆ)

#### ข้อมูลที่จัดเก็บต่อ 1 ผู้ติดต่อ

| ฟิลด์ข้อมูล    | รายละเอียด                            | ข้อมูลบังคับ |
| :------------- | :------------------------------------ | :----------: |
| ชื่อ-นามสกุล   | ชื่อเต็มของผู้ติดต่อ                  |     ใช่      |
| ความสัมพันธ์   | พ่อ / แม่ / คู่สมรส / พี่น้อง / อื่นๆ |     ใช่      |
| เบอร์โทร       | เบอร์โทรศัพท์หลักสำหรับติดต่อ         |     ใช่      |
| อีเมล          | อีเมลสำหรับติดต่อ                     |     ไม่      |
| ที่อยู่        | ที่อยู่สำหรับติดต่อ                   |     ไม่      |
| ลำดับการติดต่อ | ลำดับความสำคัญในการติดต่อ             |     ใช่      |

---

### 3.6 ข้อมูลสุขภาพ (Health Records)

> **ข้อสำคัญ:** ข้อมูลส่วนนี้จัดเป็นข้อมูลอ่อนไหว (Sensitive Data) ตาม พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล (PDPA) จะมีการควบคุมสิทธิ์การเข้าถึงเป็นพิเศษ เฉพาะฝ่ายบุคคลและผู้มีอำนาจเท่านั้น

#### ข้อมูลทั่วไป

- กรุ๊ปเลือด
- ส่วนสูง
- น้ำหนัก
- ประวัติการแพ้ยา
- ประวัติการแพ้อาหาร

#### โรคประจำตัว (เพิ่มได้หลายรายการ)

- ชื่อโรคประจำตัว
- วันที่ตรวจพบ
- ระดับความรุนแรง
- ยาที่ใช้ประจำ
- หมายเหตุเพิ่มเติม

---

### 3.7 เอกสารและใบรับรอง (Document List)

ระบบรองรับการจัดเก็บเอกสารสำคัญของพนักงาน พร้อมระบบแจ้งเตือนการต่ออายุอัตโนมัติ

#### ประเภทเอกสารที่ระบบรองรับ

- ใบรับรองความสามารถ
- ใบอนุญาตประกอบวิชาชีพ
- ประกาศนียบัตร
- สำเนาบัตรประชาชน และทะเบียนบ้าน
- หนังสือเดินทาง และวีซ่า
- เอกสารอื่นๆ (ผู้ดูแลระบบสามารถเพิ่มประเภทเอกสารใหม่ได้)

#### ข้อมูลที่จัดเก็บต่อ 1 เอกสาร

| ฟิลด์ข้อมูล    | รายละเอียด                            |
| :------------- | :------------------------------------ |
| ประเภทเอกสาร   | กลุ่มของเอกสาร                        |
| ชื่อเอกสาร     | ชื่อของเอกสาร                         |
| เลขที่เอกสาร   | เลขที่อ้างอิง                         |
| หน่วยงานที่ออก | ผู้ออกเอกสาร                          |
| วันที่ออก      | วันที่ออกเอกสาร                       |
| วันหมดอายุ     | วันที่หมดอายุ (ถ้ามี)                 |
| ตลอดชีพ        | ตัวเลือกสำหรับเอกสารที่ไม่หมดอายุ     |
| ไฟล์แนบ        | ไฟล์ PDF หรือรูปภาพ                   |
| สถานะ          | ใช้งานได้ / ใกล้หมดอายุ / หมดอายุแล้ว |

#### ระบบแจ้งเตือนการต่ออายุเอกสาร

ระบบจะตรวจสอบเอกสารที่ใกล้หมดอายุทุกวัน และแจ้งเตือนล่วงหน้าผ่านอีเมลและระบบ ในช่วงเวลาดังนี้

|      ช่วงเวลา      | ระดับการแจ้งเตือน     | ลักษณะการแจ้ง |
| :----------------: | :-------------------- | :-----------: |
| 90 วัน ก่อนหมดอายุ | แจ้งเตือนครั้งแรก     |     ปกติ      |
| 60 วัน ก่อนหมดอายุ | แจ้งเตือนครั้งที่สอง  |     ปกติ      |
| 30 วัน ก่อนหมดอายุ | แจ้งเตือนเร่งด่วน     |   เร่งด่วน    |
| 7 วัน ก่อนหมดอายุ  | แจ้งเตือนครั้งสุดท้าย |     วิกฤต     |

#### ผู้รับการแจ้งเตือน

- พนักงานเจ้าของเอกสาร
- หัวหน้างานโดยตรง (Optional)
- ฝ่ายบุคคล (Optional)

## 4. การเชื่อมโยงกับโมดูลอื่น

ข้อมูลพนักงานเป็นข้อมูลพื้นฐานที่จะเชื่อมต่อกับโมดูลอื่นในระบบ HR ทั้งหมด เพื่อให้ระบบทำงานเป็นเอกภาพ

| โมดูลที่เชื่อมโยง          | การใช้ประโยชน์                      |
| :------------------------- | :---------------------------------- |
| Time Tracking (เวลาทำงาน)  | ดึงข้อมูลพนักงานมาแสดงในตารางเวลา   |
| Leave Management (วันหยุด) | คำนวณสิทธิ์วันลาตามประเภทพนักงาน    |
| Attendance (การลงเวลา)     | เชื่อมกับการลงเวลาเข้า-ออกงาน       |
| Payroll (การจ่ายเงิน)      | ใช้ข้อมูลธนาคาร ภาษี และประกันสังคม |
