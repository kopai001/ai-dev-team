### 2.2 Modal: เพิ่ม/แก้ไขหมวด Custom

```
┌────────────────────────────────────────┐
│  เพิ่มหมวดรายการ                        │
├────────────────────────────────────────┤
│  ชื่อหมวด : [___________________]      │
│  ประเภท   : (●) รายรับ  ( ) รายหัก    │
│  สถานะ    : (●) เปิดใช้  ( ) ปิด       │
│                                        │
│  💡 เมื่อสร้างงวดใหม่ รายการที่เปิดใช้  │
│     จะแสดงเป็น column เพิ่มใน Step 1   │
│     ค่าเริ่มต้นของแต่ละช่อง = null      │
│                                        │
│         [ยกเลิก]  [บันทึก]             │
└────────────────────────────────────────┘
```

**Business Rules:**
- Default columns: แก้ชื่อได้ ปิดไม่ได้ ลบไม่ได้
- Custom columns: เปิด/ปิด/ลบได้ (ลบได้เฉพาะถ้าไม่มีงวดที่ใช้อยู่)
- column ที่ปิด → ไม่แสดงในงวดที่สร้างใหม่ (งวดเก่าไม่กระทบ)
- column ที่เปิด → แสดงใน Step 1 ทุกงวดที่สร้างหลังจากนั้น

#### Step 1: คำนวณและกรอกข้อมูล

ระบบ auto-คำนวณและเติมค่าเริ่มต้นจาก Attendance / OT / Leave modules

**UX Notes:**
- กด **[คำนวณใหม่ทั้งหมด]** → recalculate จาก module อื่น และเติมค่าใหม่ (เซลล์ที่ user เคย mark ✎ จะถามยืนยันก่อน overwrite)
- กด **[นำเข้าภาษีจากงวดก่อน]** → copy ค่า column ภาษีหัก ณ ที่จ่าย จากงวดล่าสุด ทั้งหมด
- **สุทธิ** คำนวณ live ทุกครั้งที่แก้ไขเซลล์ใดๆ ในแถวนั้น
- cell ที่ค่า = null → แสดงเป็น placeholder สีเทา "(null)" และจะไม่ปรากฏในสลิป

## 9. Business Flow สรุป (End-to-End)

```
Payroll Staff                    System                           Output
───────────────────────────────────────────────────────────────────────────

1. สร้างงวด ──────────────►  INSERT PayrollPeriod (DRAFT)
                              ดึง active custom columns จาก Master Data
                              เพื่อ attach กับงวดนี้

2. [ดำเนินการต่อ] ────────►  period.status → PROCESSING

3. [คำนวณใหม่ทั้งหมด] ────►  Batch query: Employee+OT+Attendance+Leave
   (Step 1)                   Loop พนักงาน → UPSERT PayrollRecord
                              พร้อม line items ทุกหมวด (default + custom)
                              เซลล์ที่ auto-fill จะ mark isAutoCalculated=true

4. User แก้ไขเซลล์ ───────►  PATCH PayrollRecord field
   (Step 1)                   mark isUserOverridden=true บน field นั้น

5. [ดำเนินการต่อ] ────────►  ไปยัง Step 2 (summary)

6. [ล็อกงวดและสร้างสลิป]─►  period.status → LOCKED
   (Step 3)                   PayrollRecord.status → CONFIRMED
                              ───────────────────────────────────────────
                              MONTHLY: INSERT BankTransferFile ────────► ไฟล์ Krungthai
                              DAILY:   INSERT CashPaymentDocument ─────► ใบเงินสด

7. ดาวน์โหลด + ส่งธนาคาร     UPDATE BankTransferFile.status → SUBMITTED

8. [บันทึกจ่ายแล้ว] ───────►  period.status → PAID, paidAt = วันจ่ายจริง
```
